# 📋 PROMPT MAESTRO: Aplicación de Reporte y Análisis de Incidentes Urbanos con Streamlit, FastAPI y H3

## 🎯 CONTEXTO Y OBJETIVO
Diseña e implementa una aplicación web completa en Python que permita a ciudadanos y autoridades **reportar incidentes urbanos** (hurto, violencia, conflictos de convivencia), **visualizarlos geoespacialmente** mediante la cuadrícula hexagonal `h3-py`, y **exponer una API REST** construida con FastAPI para integrar este sistema con servicios externos (alertas ciudadanas, dashboards institucionales, APIs de emergencias, etc.).

La solución debe priorizar: usabilidad, precisión geoespacial, escalabilidad, interoperabilidad y buenas prácticas de desarrollo.

---

## 🏗️ ARQUITECTURA PROPUESTA
```
[Frontend] Streamlit (UI interactiva, mapas, formularios, dashboards)
      ↓ HTTP/REST
[Backend] FastAPI (API REST, validación, lógica de negocio, orquestación)
      ↓ ORM/SQL
[Base de Datos] PostgreSQL + PostGIS o SQLite (para prototipado)
      ↓ Cálculo geoespacial
[Procesamiento H3] h3-py (indexación, agregación, neighbors, k-ring)
      ↓ HTTP Async
[Servicios Externos] FastAPI como proxy/agregador (webhooks, polling, sincro/async)
```

## 🔧 FUNCIONALIDADES DETALLADAS

### 1. Formulario de Reporte (Streamlit)
- Campos: tipo de incidente (`hurto`, `violencia`, `convivencia`), coordenadas GPS (lat/lng), dirección textual (opcional), fecha/hora, descripción breve, nivel de gravedad (1-5), carga de evidencia (imagen/audio opcional).
- Validación en cliente y servidor: rangos lat/lng válidos, campos obligatorios, sanitización de texto.
- Geocodificación inversa opcional para mostrar dirección legible.

### 2. Indexación H3 Automática
- Al guardar, calcular automáticamente el índice H3 (resolución 7 u 8 configurable vía `.env`).
- Almacenar: `h3_index`, `h3_center_lat`, `h3_center_lng`, `resolution`.
- Permitir consulta por `k-ring` (vecinos hexagonales) para análisis de densidad.

### 3. Visualización Interactiva
- Mapa con hexágonos coloreados por:
  - Densidad de incidentes
  - Tipo predominante
  - Temporalidad (filtro por rango de fechas)
- Capas superponibles: límites administrativos, zonas de alto riesgo, reportes recientes.
- Tooltips con métricas por hexágono: conteo, tipos, fechas, tendencia.

### 4. Dashboard Analítico
- KPIs: total reportes, distribución por tipo, picos horarios/diarios, hexágonos críticos.
- Gráficas: series temporales, distribución geográfica, comparativa por tipo/gravedad.
- Exportación: CSV/JSON filtrado.

### 5. Integración con Servicios Externos (FastAPI)
- Endpoints para recibir/enviar datos a APIs de terceros (alertas SMS, CRM municipal, sistemas de emergencia).
- Patrón: cola asíncrona con `httpx` + reintentos exponenciales + logging estructurado.
- Webhooks configurables vía UI o `.env`.

## 🌐 DISEÑO DE LA API REST (FastAPI)

| Método | Ruta | Descripción | Auth |
|--------|------|-------------|------|
| `POST` | `/api/v1/incidents` | Crear reporte | API Key / JWT |
| `GET` | `/api/v1/incidents` | Listar con filtros (tipo, fecha, h3_index, bbox) | Opcional |
| `GET` | `/api/v1/incidents/{id}` | Detalle | Opcional |
| `PUT` | `/api/v1/incidents/{id}` | Actualizar | JWT |
| `DELETE` | `/api/v1/incidents/{id}` | Eliminar/Anular | JWT |
| `GET` | `/api/v1/h3/{index}/density` | Agregación por hexágono + k-ring | API Key |
| `POST` | `/api/v1/external/sync` | Trigger sincronización con servicio externo | JWT |
| `GET` | `/api/v1/health` | Estado del servicio | Pública |

**Requisitos API:**
- Documentación OpenAPI/Swagger automática (`/docs`)
- Validación estricta con Pydantic v2
- Rate limiting básico (`slowapi` o middleware custom)
- Respuestas estandarizadas: `{ "status": "ok", "data": {...}, "meta": {...} }`
- Manejo de errores con `HTTPException` y logs estructurados

## 🗺️ INTEGRACIÓN CON H3 Y VISUALIZACIÓN

- Usar `h3.latlng_to_cell(lat, lng, resolution)` para asignar índice.
- Generar hexágonos para mapa con `h3.cell_to_boundary(index)` → convertir a GeoJSON.
- Agrupar incidentes con `pandas` + `h3` y calcular métricas por celda.
- Cache de resultados H3 con `functools.lru_cache` o Redis (si escala).
- Resolución recomendada: `7` (manzanas/barrios) o `8` (cuadras). Configurable.

---

## 🔒 SEGURIDAD, VALIDACIÓN Y RENDIMIENTO

- **Autenticación:** JWT para usuarios registrados, API Keys para servicios externos.
- **Autorización:** Roles (`ciudadano`, `operador`, `admin`).
- **Validación:** Pydantic schemas + sanitización de inputs + prevención XSS/SQLi.
- **Privacidad:** Anonimización opcional de ubicación (jitter ±50m) antes de exposición pública.
- **Rendimiento:** 
  - Consultas H3 precalculadas o materializadas.
  - `async/await` en FastAPI para I/O.
  - Paginación y filtros indexados en DB.
  - Cache de capas de mapa en Streamlit (`@st.cache_data`).
